# 深いモジュール

「A Philosophy of Software Design」より:

**深いモジュール** = 小さなインターフェース + 多くの実装

```
┌─────────────────────┐
│ 小さなインターフェース │  ← 少ないメソッド、単純な引数
├─────────────────────┤
│                     │
│                     │
│    深い実装          │  ← 複雑なロジックを隠す
│                     │
│                     │
└─────────────────────┘
```

**浅いモジュール** = 大きなインターフェース + 少ない実装。避けること。

```
┌─────────────────────────────────┐
│      大きなインターフェース        │  ← 多いメソッド、複雑な引数
├─────────────────────────────────┤
│       薄い実装                   │  ← ほぼ素通し
└─────────────────────────────────┘
```

インターフェースを設計するときに尋ねること:

- メソッド数を減らせるか？
- 引数を単純にできるか？
- より多くの複雑さを内部に隠せるか？
