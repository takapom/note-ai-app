# 言語

このスキルが行うすべての提案で共有する語彙。これらの用語を厳密に使うこと。「コンポーネント」「サービス」「API」「境界」で置き換えない。一貫した言葉遣いこそが要点である。

## 用語

**モジュール**
インターフェースと実装を持つものすべて。意図的にスケール非依存にしている。関数、クラス、パッケージ、tier をまたぐスライスにも等しく適用する。
_避ける_: unit、component、service。

**インターフェース**
caller がモジュールを正しく使うために知る必要があるものすべて。型シグネチャに加え、不変条件、順序制約、エラーモード、必要な設定、性能特性を含む。
_避ける_: API、signature。狭すぎる。これらは型レベルの表面だけを指す。

**実装**
モジュールの内側にあるもの。つまりコード本体。**アダプター**とは区別する。小さなアダプターが大きな実装を持つこともある。例: Postgres repo。大きなアダプターが小さな実装を持つこともある。例: in-memory fake。継ぎ目が話題なら「アダプター」を使い、それ以外では「実装」を使う。

**深さ**
インターフェース上のレバレッジ。caller や test が、学ぶ必要のあるインターフェース 1 単位あたりに、どれだけの振る舞いを実行できるか。小さなインターフェースの背後に多くの振る舞いがあるとき、モジュールは**深い**。インターフェースが実装とほぼ同じくらい複雑なとき、モジュールは**浅い**。

**継ぎ目** _(Michael Feathers より)_
その場所を編集せずに振る舞いを変えられる場所。モジュールのインターフェースが存在する**位置**。継ぎ目をどこに置くかは、それ自体が設計判断であり、その背後に何を置くかとは別である。
_避ける_: boundary。DDD の bounded context と重なりすぎる。

**アダプター**
継ぎ目でインターフェースを満たす具体物。中身が何かではなく、どの slot を埋めるかという**役割**を表す。

**レバレッジ**
深さによって caller が得るもの。学ぶ必要のあるインターフェース 1 単位あたりに得られる能力が増える。1 つの実装が、N 個の call site と M 個の test へ返ってくる。

**局所性**
深さによって maintainer が得るもの。変更、バグ、知識、検証が caller に分散せず、一箇所に集中する。一度直せば、すべて直る。

## 原則

- **深さは実装ではなく、インターフェースの性質である。** 深いモジュールは、内部では小さく mock 可能で交換可能な部品から構成されていてよい。ただし、それらはインターフェースの一部ではない。モジュールは、インターフェース上の**外部の継ぎ目**に加えて、**内部の継ぎ目**を持てる。内部の継ぎ目は実装に閉じており、自身のテストで使われる。
- **削除テスト。** そのモジュールを削除することを想像する。複雑さが消えるなら、モジュールは何も隠していなかった。つまり通過点だった。複雑さが N 個の caller に再出現するなら、モジュールは役割を果たしていた。
- **インターフェースがテスト面である。** caller と test は同じ継ぎ目を越える。インターフェースの先をテストしたくなるなら、そのモジュールはおそらく形が悪い。
- **アダプター 1 つは仮説上の継ぎ目。アダプター 2 つは実在する継ぎ目。** 実際にそこで変化するものがない限り、継ぎ目を導入しない。

## 関係

- **モジュール**はちょうど 1 つの**インターフェース**を持つ。caller と test に提示する表面である。
- **深さ**は、**インターフェース**に照らして測られる**モジュール**の性質である。
- **継ぎ目**は、**モジュール**の**インターフェース**が存在する場所である。
- **アダプター**は**継ぎ目**に置かれ、**インターフェース**を満たす。
- **深さ**は caller に**レバレッジ**を、maintainer に**局所性**を生む。

## 却下する捉え方

- **実装行数とインターフェース行数の比率としての深さ** (Ousterhout): 実装を水増しすることを報酬にしてしまう。このスキルでは、深さをレバレッジとして扱う。
- **TypeScript の `interface` keyword や class の public methods としての「インターフェース」**: 狭すぎる。ここでのインターフェースは、caller が知る必要のあるすべての事実を含む。
- **"Boundary"**: DDD の bounded context と重なりすぎる。**継ぎ目**または**インターフェース**と言う。
