# Codex タスク化ガイド

ドキュメント種別: ガイド。このガイドはポリシーを定義しません。`docs/contracts/superset-codex-workflow.md` を実装します。

## 標準タスク形式

次を使用します:

- 目的
- コンテキスト
- 制約
- 実装メモ
- 完了条件
- 検証

## 計画優先タスク

次の場合は計画優先を使用します:

- データモデルの変更。
- 操作ルーター の変更。
- コンテキスト組み立て。
- スケジューラー/ライフサイクルの変更。
- ランタイム/プロバイダー抽象。
- editor architecture。
- Turso migration。
- Cloudflare Agents design。
- トポロジーに影響するリファクタリング。

## タスクレビューのチェックリスト

Codex を起動する前に:

1. 所有者コントラクトを明記します。
2. 検証レーンを明記します。
3. 変更されるレイヤーを明記します。
4. 非目標を明記します。
5. 実装がコントラクトを変更してよいかを明記します。
6. 変更してよい場合は、生成レジスターの更新を必須にします。
7. MVP 完了条件に関係する場合は `docs/contracts/mvp-acceptance.md` の該当項目を明記します。

## プロンプトの雛形

`.codex/prompts/task-template.md` を参照してください。
