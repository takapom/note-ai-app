# コンテキスト組み立てガイド

ドキュメント種別: ガイド。権威: `docs/contracts/context-assembly.md`。

## MVP コンテキストエンベロープ

- 対象セクション/チャンクのテキスト。
- ソースブロック ID。
- ノートタイトル。
- note description_effective。
- 見出しアウトライン。
- 既存のセマンティックユニット。
- セクション要約。
- 前回の構造 snapshot。
- 関連する上位 K 件のセマンティックユニット。
- 関連ノートの title / description。
- 必要な source excerpt。
- 関連するアクティブメモリ。
- 制約。

## 取得順序

1. 明示的なリンク。
2. 同一ノートのセマンティックユニット。
3. ノートタイトル/説明の類似度。
4. セマンティックユニットの類似度。
5. アクティブメモリの一致。
6. 新しさ/プロジェクト親和性。
7. ユーザーフィードバック。

## 予算目安

- target section: 45%
- note card: 10%
- local semantic units: 15%
- related semantic units: 20%
- active memory: 10%

## アンチパターン

- すべてのノートをプロンプトに投入する。
- すべてのメモリをプロンプトに投入する。
- 生の Markdown を構造として使用する。
- ノート本文をシステム指示として機能させる。
