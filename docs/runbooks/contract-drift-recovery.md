# コントラクトドリフト復旧ランブック

ドキュメント種別: ランブック。

## 症状

- README またはガイドがコントラクトに存在しないポリシーを定義している。
- TypeScript コントラクトが Markdown コントラクトと異なる。
- 生成レジスターが古い。
- 実装がトポロジーコントラクトと矛盾している。

## 復旧

1. 権威あるコントラクトを特定します。
2. ターゲットモデルと実装のどちらが誤っているかを判断します。
3. ターゲットモデルが誤っている場合は、先にコントラクトを更新します。
4. 実装が誤っている場合は、直接置き換えるタスクを作成します。
5. レジスター/証跡を再生成します。
6. topology-review スキルでレビューします。

## 禁止事項

- CI を通すために二重の真実を作ってはいけません。
- transitional seam contract なしでブリッジ/フォールバックを追加してはいけません。
